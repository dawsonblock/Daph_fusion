# Legacy v2.5 Artifacts (Frozen)

These artifacts were generated by `scripts/run_enhanced_experiment.py` before
the experimental-truth repair (v2.6). They are preserved as baselines but are
**not** valid representations of the algorithms they claim to evaluate.

## Relabeled method names

The following method labels in the legacy artifacts are **scientifically
incorrect** and must not be cited as validation of the named algorithm:

| Legacy label | Actual implementation | Correct label |
|---|---|---|
| `Fisher` | `\|delta\|^2`-weighted merge (squared task-vector magnitude) | `delta_squared_weighting_legacy` |
| `ExFusion` | DARE → `\|delta\|^2`-weighted merge → TIES on single vector (no cross-expert election) | `dare_delta_squared_trim_legacy` |
| `AGX` | Hand-written sign-conflict heuristic (not LayerwiseGeometrySearchEngine) | `heuristic_merge_policy_legacy` |

## Why these are preserved

- `task_arithmetic`, `mean_merge`, `DARE`, `TIES`, `DARE_TIES`, and
  `weighted_task_arithmetic` results are legitimate (the implementations match
  the algorithm names).
- The legacy Fisher/ExFusion/AGX numbers are useful as baselines for the
  squared-magnitude-weighting heuristic, but must not be presented as empirical
  Fisher, full ExFusion, or AGX search results.

## Bootstrap CI caveat

The 95% CIs in `method_statistics.json` are bootstrapped over 5 seed-level
aggregate retention values, not over per-sample evaluation data. For
deterministic methods (TIES, task_arithmetic, etc.) all 5 seed values are
identical, producing degenerate intervals like `[0.8155, 0.8155]`. These CIs
do **not** measure evaluation uncertainty.
